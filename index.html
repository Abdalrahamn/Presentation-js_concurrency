What is a Callback?

Definition

A function passed as an argument to another function, to be executed later

Simple Callback Example

function greet(name, callback) {
  console.log('Hello ' + name);
  callback();
}

function sayGoodbye() {
  console.log('Goodbye!');
}

greet('Alice', sayGoodbye);
// Output: Hello Alice
// Output: Goodbye!

Array Methods with Callbacks

const numbers = [1, 2, 3, 4, 5];

// map() uses a callback to transform each element
const doubled = numbers.map(function(num) {
  return num * 2;
});

// filter() uses a callback to test each element
const evens = numbers.filter(function(num) {
  return num % 2 === 0;
});

console.log(doubled); // [2, 4, 6, 8, 10]
console.log(evens);   // [2, 4]

Synchronous Callbacks

Executed immediately in the same thread

function processData(data, callback) {
  const result = data.toUpperCase();
  callback(result); // Executes immediately
}

processData('hello', function(result) {
  console.log(result); // HELLO
});

Asynchronous Callbacks

Executed later when an operation completes

function fetchData(callback) {
  setTimeout(() => {
    const data = 'Server response';
    callback(data); // Executes after 1 second
  }, 1000);
}

fetchData(function(data) {
  console.log(data); // Server response
});

Debouncing

Wait for pause in events

Only executes after events stop

function debounce(func, delay) {
  let timeoutId;
  return function(...args) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => 
      func.apply(this, args), delay);
  };
}

// Usage: Search input
const debouncedSearch = debounce(search, 300);

Perfect for:

  • Search input
  • Resize events
  • API calls

AbortController - Modern Cancellation

Cancel Operations

Modern way to cancel fetch requests, timeouts, and other async operations

Prevent Memory Leaks

Avoid callbacks from completed operations affecting unmounted components

Better UX

Cancel slow requests when user navigates away or changes input

Basic Usage

// Create controller
const controller = new AbortController();
const signal = controller.signal;

// Use with fetch
fetch('/api/data', { signal })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => {
    if (error.name === 'AbortError') {
      console.log('Request was cancelled');
    }
  });

// Cancel the request
controller.abort();

AbortController with Timeout

Auto-cancel after timeout

async function fetchWithTimeout(url, timeout = 5000) {
  const controller = new AbortController();
  setTimeout(() => controller.abort(), timeout);
  
  try {
    const response = await fetch(url, { 
      signal: controller.signal 
    });
    return await response.json();
  } catch (error) {
    if (error.name === 'AbortError') {
      throw new Error('Request timeout');
    }
    throw error;
  }
}

Best Practices

Do

  • Use async/await for readability
  • Handle errors with try/catch
  • Use Promise.all for parallel ops
  • Avoid blocking the main thread

Don't

  • Mix callbacks with promises
  • Forget error handling
  • Use sync ops in loops
  • Create unnecessary promises
